# 1.3 시장의 문제점 (Market Problems)

## 1.3.1 P2P 거래 시장 현황

### 시장 규모
- **글로벌 P2P 결제 시장:** 2024년 기준 약 3,500억 달러
- **연평균 성장률:** 15.2% (2024-2029 예상)
- **주요 성장 동력:** 디지털 경제 확산, 모바일 결제 증가

### 주요 플레이어
| 플랫폼 | 시장 점유율 | 주요 문제점 |
|--------|-------------|-------------|
| PayPal | 25% | 높은 수수료, 제한적 글로벌 접근 |
| Venmo | 15% | 미국 중심, 보안 취약점 |
| Zelle | 12% | 은행 계좌 필수, 사기 대응 부족 |
| 기타 | 48% | 파편화된 서비스, 상호 호환성 부족 |

## 1.3.2 핵심 문제점 분석

### 1. 사기 및 보안 위험

#### 통계 데이터
- **연간 P2P 사기 피해액:** 약 100억 달러 (2023년 기준)
- **사기 신고 건수:** 전년 대비 30% 증가
- **피해 회복률:** 평균 15% 미만

#### 주요 사기 유형
1. **가짜 판매자 사기**
   - 상품 미배송 후 잠적
   - 가짜 상품 판매
   - 허위 정보 제공

2. **결제 후 취소 사기**
   - 상품 수령 후 결제 취소 요청
   - 신용카드 차지백 악용
   - 계정 해킹 후 거래 부인

3. **피싱 및 계정 탈취**
   - 가짜 결제 페이지 생성
   - 개인정보 수집 후 계정 도용
   - 소셜 엔지니어링 공격

### 2. 높은 거래 수수료

#### 수수료 구조 분석
| 거래 유형 | 기존 플랫폼 평균 | 사용자 부담 |
|-----------|------------------|-------------|
| 국내 송금 | 2.9% + $0.30 | 높음 |
| 국제 송금 | 4.5% + $5.00 | 매우 높음 |
| 상품 결제 | 3.5% + $0.49 | 높음 |
| 서비스 결제 | 2.9% + $0.30 | 보통 |

#### 수수료 부담의 영향
- **소액 거래 기피:** 수수료가 거래액 대비 과도
- **소상공인 부담:** 마진 압박으로 인한 경영 어려움
- **소비자 전가:** 최종적으로 소비자가 부담

### 3. 신뢰 부재 문제

#### 신뢰도 조사 결과
- **거래 전 불안감:** 사용자의 78%가 경험
- **상대방 신뢰도 확인 어려움:** 85%가 문제로 인식
- **분쟁 해결 만족도:** 평균 2.3/5.0점

#### 신뢰 부재의 원인
1. **정보 비대칭**
   - 거래 상대방 정보 부족
   - 과거 거래 이력 확인 불가
   - 평판 시스템의 조작 가능성

2. **중재 시스템 부재**
   - 분쟁 발생 시 해결 방법 부족
   - 플랫폼별 상이한 정책
   - 긴 분쟁 해결 시간

## 1.3.3 기술적 한계

### 1. 실시간 위험 분석 부족
- **사후 대응 중심:** 사기 발생 후 대응
- **패턴 인식 한계:** 새로운 사기 수법 대응 지연
- **수동적 모니터링:** 인력 의존적 감시 체계

### 2. 플랫폼 간 호환성 부족
- **파편화된 생태계:** 플랫폼별 독립적 운영
- **데이터 공유 제한:** 위험 정보 공유 부족
- **표준화 부재:** 통일된 보안 기준 없음

### 3. 확장성 문제
- **처리 용량 한계:** 대량 거래 시 지연 발생
- **지역별 제한:** 국가별 규제로 인한 서비스 제한
- **기술 부채:** 레거시 시스템의 한계

## 1.3.4 규제 및 컴플라이언스 이슈

### 규제 환경의 복잡성
- **국가별 상이한 규제:** 글로벌 서비스 제공 어려움
- **KYC/AML 요구사항:** 복잡한 신원 확인 절차
- **데이터 보호 규정:** GDPR 등 개인정보 보호 강화

### 컴플라이언스 비용
- **규제 준수 비용:** 연간 수익의 10-15%
- **법무 비용:** 지속적인 법적 검토 필요
- **시스템 업데이트:** 규제 변화에 따른 시스템 개선

## 1.3.5 사용자 경험 문제

### 복잡한 사용 절차
1. **계정 생성 및 인증**
   - 복잡한 신원 확인 과정
   - 다수의 서류 제출 요구
   - 긴 승인 대기 시간

2. **거래 과정의 복잡성**
   - 다단계 인증 절차
   - 복잡한 수수료 구조
   - 불명확한 거래 상태

### 고객 지원 한계
- **응답 시간 지연:** 평균 24-48시간
- **언어 장벽:** 제한적 다국어 지원
- **해결 능력 부족:** 기술적 문제 해결 한계

## 1.3.6 시장 기회 분석

### 해결되지 않은 니즈
1. **실시간 위험 평가 시스템**
2. **투명하고 공정한 수수료 구조**
3. **글로벌 통합 결제 플랫폼**
4. **AI 기반 자동화된 분쟁 해결**

### 기술 발전 기회
- **AI/ML 기술 성숙:** 정확한 위험 예측 가능
- **블록체인 기술 발전:** 투명하고 안전한 거래 기록
- **모바일 기술 발전:** 편리한 사용자 경험 제공
