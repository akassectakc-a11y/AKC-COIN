# 📋 AKASSECT 문서 유형 가이드

## 🎯 **명확한 구분: 웹페이지 vs PDF 백서**

### 📱 **웹페이지 뷰어** (index.html)
```
목적: 온라인 탐색 및 미리보기
형태: 인터랙티브 웹사이트
사용자: 웹사이트 방문자, 빠른 정보 확인
특징:
- 반응형 디자인
- 네비게이션 메뉴
- 섹션별 개별 로딩
- 검색 및 필터 기능
- 소셜 공유 기능
```

### 📄 **PDF 백서** (AKASSECT-COMPLETE-WHITEPAPER.html)
```
목적: 공식 문서 배포 및 제출
형태: A4 페이지 연속 문서
사용자: 투자자, 거래소, 규제기관
특징:
- A4 페이지 형식 (@page 규칙)
- 인쇄 최적화
- 연속적 읽기 경험
- 페이지 번호 및 푸터
- 목차 및 참조
```

## 🚀 **사용 시나리오**

### 📱 **웹페이지 사용 시나리오**
1. **웹사이트 방문자**
   - 프로젝트 개요 빠른 확인
   - 특정 섹션만 선택적 읽기
   - 소셜 미디어 공유

2. **개발자 및 기술진**
   - 기술 문서 검색
   - API 문서 참조
   - 코드 예제 확인

### 📄 **PDF 백서 사용 시나리오**
1. **투자자 및 기관**
   - 공식 문서로 다운로드
   - 인쇄하여 오프라인 검토
   - 법적 검토 및 분석

2. **거래소 제출**
   - 상장 신청서 첨부
   - 규제 기관 제출
   - 공식 아카이브

3. **미디어 및 언론**
   - 보도자료 첨부
   - 공식 레퍼런스 문서
   - 인용 및 참조

## 🎯 **최종 결론**

**웹페이지**와 **PDF 백서**는 **완전히 다른 목적**을 가진 별개의 문서입니다:

- **웹페이지**: 온라인 마케팅 및 정보 제공
- **PDF 백서**: 공식 문서 및 기관 제출용

두 형태 모두 **동일한 내용**을 담고 있지만, **형식과 사용 목적**이 다르므로 각각의 특성에 맞게 최적화되어야 합니다.
