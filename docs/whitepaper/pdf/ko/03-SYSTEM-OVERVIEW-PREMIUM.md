### 🚀 실시간 거래 처리 플로우차트

#### 시나리오 1: 표준 P2P 거래 (성공 케이스)

```mermaid
sequenceDiagram
    participant U as 👤 사용자
    participant AI as 🤖 AI Engine
    participant SC as 📜 Smart Contract
    participant BC as ⛓️ Blockchain
    participant N as 🌐 Network
    
    Note over U,N: 🎯 거래 시작 단계
    U->>AI: 거래 요청 ($500)
    activate AI
    AI->>AI: 실시간 위험 분석
    Note right of AI: • 지갑 신뢰도 검증<br/>• 거래 패턴 분석<br/>• 사기 확률 계산
    AI->>U: ✅ 안전 등급 (85점)
    deactivate AI
    
    Note over U,N: 💰 에스크로 예치 단계
    U->>SC: $500 AKC 예치
    activate SC
    SC->>BC: 에스크로 컨트랙트 생성
    BC->>N: 거래 기록 전파
    N->>SC: 블록 확인 완료
    SC->>U: 예치 완료 알림
    deactivate SC
    
    Note over U,N: 🔍 모니터링 단계
    loop 실시간 모니터링
        AI->>BC: 거래 상태 확인
        AI->>AI: 이상 징후 탐지
    end
    
    Note over U,N: ✅ 거래 완료 단계
    U->>SC: 거래 완료 확인
    activate SC
    SC->>AI: 최종 검증 요청
    AI->>SC: 승인 신호
    SC->>BC: 자동 정산 실행
    BC->>N: 완료 기록 저장
    SC->>U: 거래 완료 알림
    deactivate SC
```

```mermaid
flowchart TD
    A[👤 거래 요청] --> B{🤖 AI 위험 분석}
    B -->|위험도 < 60점| C[🚨 위험 거래 탐지]
    B -->|위험도 ≥ 60점| D[✅ 안전 거래 승인]
```
